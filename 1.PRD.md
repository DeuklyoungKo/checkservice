# PRD (Product Requirement Document) - Trend Intelligence

## 1. 프로젝트 개요 및 목표
본 프로젝트의 최우선 목표는 **'수익성 있는 서비스 아이디어를 발굴하고 도출하는 것'**입니다. AI를 활용해 시장의 결핍을 데이터 기반으로 찾아내고, 실제 서비스 개발 후 광고를 통한 유입량 측정을 통해 비즈니스의 지속 가능성을 사용자가 직접 판단할 수 있도록 돕는 시스템을 구축합니다.

- **핵심 목표**: 커뮤니티 데이터와 사용자 입력을 분석하여 PUFE 프레임워크를 기반으로 '실제 해결 가치가 높은 페인포인트'를 발굴하고 수익화 가이드를 제공함.
- **AI의 역할**:
  1. **통계적 니즈 및 PUFE 도출**: 커뮤니티 데이터에서 통계적 유의미성(빈도, 상승률)을 추출하고, 이를 기반으로 PUFE(Pain, Urgency, Frequency, Existing Solution) 스코어를 산출함.
  2. **사용자 아이디어 분석**: 사용자가 직접 입력한 결핍 상황을 PUFE 프레임워크로 진단하고 3가지 유형의 해결 솔루션을 제안함.
  3. **해결 가이드 제공**: 도출된 문제에 대해 기술적 구현 및 마케팅 전략(GTM)을 포함한 실행 가이드를 제시함.

## 2. 전략적 포지셔닝 및 차별화 (Pivot Strategy)
기존 글로벌 트렌드 서비스(Exploding Topics 등)와의 차별화를 위해 **"Global Data, Local Insight"** 전략을 채택합니다.

- **Bottom-up Discovery**: 단순 트렌드 정보를 넘어 "대중이 반복적으로 호소하는 페인포인트"를 PUFE 지표로 객관화하여 제공함.
- **Hyper-Localization (핵심 경쟁력)**: 글로벌 트렌드를 한국적 상황에 맞춰 재해석하며, 특히 한국형 비즈니스 복제/변형 전략을 제안하는 '지능형 에이전트'로 포지셔닝함. **이를 실현하기 위해 글로벌 소스와 한국 소스를 병행 수집하는 듀얼 파이프라인을 구축한다.** (상세: §4-A 참조)
- **Stats-Only Principle**: 본문 내용 대신 언급 횟수, 성장률 등 통계 데이터 중심으로 정보를 노출하여 데이터의 투명성과 법적 안전성을 확보함.
- **3. 핵심 기능 (Main Features)**
    - **통계 기반 트렌드 분석 (Stats-Only)**: 본문 수집 대신 수치(Upvotes, 성장률) 중심의 PUFE 분석 리포트 제공.
    - **수익화 실행 가속 4종 패키지 (Paid Value)**:
        - **Lean Canvas**: 한 페이지로 정리된 비즈니스 구조 리포트.
        - **GTM 광고 카피**: 타겟 고객별 맞춤형 인스타그램/뉴스레터 문구 3~5종.
        - **채널 진입 가이드**: 한국 내 초기 고객 확보(GTM)를 위한 커뮤니티 및 채널 리스트.
        - **Vibe Coding 서비스 소개서**: AI 코딩 도구(Cursor 등)에 바로 입력 가능한 `.md` 형태의 상세 기획 문서.
    - **개인 비즈니스 워크스페이스**: 시스템 추천 트렌드와 내가 직접 분석한 아이디어를 영구 보관하고 관리하는 개인 허브.

## 3. 타겟 시장 및 진입 단계 (Market Phase)
자원의 효율적 배분을 위해 단계별로 시장을 확장합니다.

- **PHASE 1 (현재)**: **한국 시장 선점**. 한국인 창업자, 1인 개발자, 프리랜서를 타겟으로 '글로벌 트렌드 → 한국형 이식' 가이드를 제공하여 수익성을 검증함.
- **PHASE 2 (미래)**: **글로벌 확장**. 한국 시장에서 검증된 '니즈 도출 및 가이드 생성' 알고리즘을 영어권 및 타 아시아 국가 시장으로 수평 확장함.

## 4. 핵심 기능 및 질문에 대한 해답
사람들이 필요로 하는 서비스가 무엇인지 검색해서 도출해 내는 프로세스를 정의합니다.

- **1차 목표 (니즈 도출)**: 글로벌 RSS + 한국 소스의 통계를 교차 비교하여 PUFE 스코어를 산출하고, '한국 시장에 실제로 수요가 있는지'를 검증된 데이터로 제시함.
- **사용자 검증**: 트렌드 리포트를 바탕으로 페이지 제작 및 광고 집행을 통해 유입량 지표를 확보하여 최종 지속성을 판단함.

## 4-A. 한국 소스 강화 전략 (Korea Data Pipeline)
"Global Data, Local Insight"의 핵심 경쟁력을 실현하기 위해 현재 영어권 RSS 위주인 데이터 파이프라인에 한국 소스를 단계적으로 추가합니다.

### 문제 현황
현재 수집 소스(Reddit, Product Hunt, HN, Dev.to 등)는 전부 영어권으로, AI가 한국형 이식 가이드를 생성해도 한국 시장에 실제 수요가 있는지 검증할 데이터가 없습니다. 한국 소스가 없으면 "한국형 이식" 분석은 추측에 불과합니다.

### 단계별 추가 계획

**1단계 - 즉시 추가 가능 (무료 공식 API 활용)**
| 소스 | 수집 방법 | 확보 데이터 |
|------|-----------|------------|
| **네이버 DataLab API** | 공식 API (무료) | 키워드 월간 검색량, 연령/성별/기기 분포 |
| **GeekNews RSS** (`news.hada.io`) | RSS 피드 직접 수집 | 한국 개발자 커뮤니티 인기 토픽 |

**2단계 - 단기 추가 (경량 스크래핑)**
| 소스 | 수집 방법 | 확보 데이터 |
|------|-----------|------------|
| **크몽 카테고리별 인기 의뢰** | HTML 파싱 | 현재 한국 시장에서 돈을 내고 사는 서비스 |
| **네이버 카페/블로그 키워드** | DataLab 검색어 트렌드 API | 커뮤니티 검색 급증 키워드 |

**3단계 - 중기 추가 (고난도)**
| 소스 | 수집 방법 | 확보 데이터 |
|------|-----------|------------|
| **블라인드 반복 불만** | 키워드 모니터링 (비공식 접근 어려움) | 직장인 페인포인트 |
| **혁신의숲 / 스타트업 투자 소식** | RSS/HTML 파싱 | 투자받은 한국 스타트업 카테고리 트렌드 |

### AI 분석 연동 방식
글로벌 트렌드 키워드를 수집한 후, **네이버 DataLab API로 한국 내 검색 수요를 교차 검증**하여 두 가지 지표를 함께 PUFE 분석 프롬프트에 주입합니다:
```
[글로벌] HN 언급 횟수: 342 / Product Hunt 업보트: 1,200
[한국]   네이버 월 검색량: 8,900 / 최근 3개월 성장률: +34%
```
이를 통해 AI가 "이 트렌드가 한국에서도 실제로 검색되고 있다"는 데이터 기반 판단을 내릴 수 있게 됩니다.

## 5. 비즈니스 검증 프레임워크 (Customer Needs Verification)
본 프로젝트는 코드를 짜기 전, 아래 4단계의 '비즈니스 플레이북'을 통해 시장의 니즈를 선검증합니다.

1. **정량적 트래픽 분석 (검색 데이터 기반)**: Google Trends 및 블랙키위 등을 활용하여 키워드의 '월간 검색량'과 '연관 검색어'를 분석, 대중이 겪는 구체적인 문제를 정의합니다.
2. **경쟁사 및 커뮤니티 페인포인트 크롤링**: 크몽, 클래스101, 블라인드 등에서 경쟁 상품의 '낮은 평점 리뷰'와 반복되는 질문을 수집하여 핵심 셀링 포인트를 도출합니다.
3. **프리세일즈 (Fake Door Test)**: 상세페이지(랜딩 페이지)를 먼저 제작하고 [사전예약] 또는 [결제하기] 버튼의 CVR(전환율)을 측정하여 실제 결제 의사를 검증합니다.
4. **마이크로 타겟팅과 페르소나 딥다이브**: 타겟을 극단적으로 좁혀(예: '1년 차 프론트엔드 개발자') 그들의 시간을 정확히 몇 시간 아껴줄 수 있는지 숫자로 제안합니다.

## 6. 수익화 및 성장 전략
- **B2B 타겟 제휴 수익**: 구글 애드센스와 같은 일반 디스플레이 광고는 서비스의 신뢰도 저하 및 사용자 이탈 우려로 지양. 대신 개발자/창업자 타겟에 맞는 고단가 **제휴 마케팅(Affiliate)** 링크(예: 서버 호스팅, 도메인 등)를 활용.
- **Lean 시장성 검증**: 복잡한 메일링 기능 구축을 배제하고, 초기에는 리포트 일부분만 공개한 상태에서 가치에 공감하는 잠재 고객들의 피드백이나 비즈니스 문의(Contact)를 받을 수 있는 단순 채널(이메일 발송 링크 폼 등)만 열어두어 최소 리소스로 수요를 탐색함.
- **유료화 결제 (Polar)**: 글로벌/북미 타겟 오픈소스 PG인 **Polar**를 연동하여 특정 리포트당 마이크로 결제($1~5)를 유도. 고가의 정기 구독보다 결제 허들을 낮춰 시장성을 빠르게 검증.
- **리텐션(Retention)**: 단순 정보 열람을 넘어 유저가 아이디어를 검증하고 관리하는 '워크스페이스' 기능을 제공하여 지속적인 방문을 유도함.

## 7. 운영 및 도메인 전략
- **초기 MVP 검증 단계**: Vercel을 통해 모회사 도메인의 서브도메인(`trend.gonsuit.com`)으로 서비스 런칭. 도메인 유지비 및 관리 리소스를 0으로 유지하고 본진 사이트(`gonsuit.com`)의 브랜드 및 SEO 후광 효과(크로스셀링)를 적극 활용함.
- **성장 및 독립 단계**: 전환율 및 마이크로 결제 등 유의미한 지표 달성 시, 전용 독립 도메인(`trendscouter.com` 등)으로 완전 스핀오프(Spin-off) 분리 독립.
- **마이그레이션 리스크 관리**: 향후 독립 도메인 이전 시 트래픽 및 권위(DA) 유실 방지를 위해 기존 서브도메인에 대한 **301 영구 리다이렉션(Permanent Redirect)** 세팅 및 Supabase 인증 설정(Auth URL) 분리 연동 작업을 필수 적용함.

## [PHASE 2 - 차후 개발 예정] 아이디어 컨버터 위저드 (Idea Converter Wizard)

### 개요
사용자가 직접 자신의 아이디어·페인포인트를 입력하면, AI가 즉시 PUFE 프레임워크로 수익성을 분석하고 실행 가이드를 제공하는 도구.

### 핵심 기능 (차후 구현)
- **실시간 PUFE 분석**: 사용자 입력 텍스트 → Gemini AI → PUFE 스코어 (100점 만점) 산출
- **3단계 솔루션 제안**: 자동화 / 개선 / 혁신 유형으로 해결 전략 분기
- **부스터 패키지 (유료)**: 린 캔버스, GTM 카피, 채널 가이드, Vibe Coding 기획서(.md) 자동 생성
- **워크스페이스 연동**: 분석 결과를 `user_id`와 연동하여 DB에 저장 및 이력 관리
- **결제 잠금 해제**: Polar Checkout 연동 ($3) → 웹훅으로 즉시 블러 해제

### 진행 조건 (트리거)

## 🔄 GitHub Actions 기반 데이터 수집 방식

"데이터 스크롤(수집)"은 **GitHub Actions**로 자동화되어 있습니다.

### 파일: [trend-collect.yml](file:///d:\Work_Gon\260303_TrendScouter\.github\workflows\trend-collect.yml)

```
GitHub Actions Scheduler
       │
       ▼ 매시간 정각 (cron: '0 * * * *')
  ubuntu-latest 실행환경
       │
       ▼ node scripts/rss-collector.js 실행
  RSS 피드 6개 순회 수집
   ├─ indie-hackers (HN RSS)
   ├─ reddit-sideproject
   ├─ product-hunt
   ├─ hacker-news (HN RSS)
   ├─ dev.to
   └─ zdnet-korea
       │
       ▼ 키워드 점수 계산 (impactScore >= 10만 통과)
       │
       ▼ Gemini AI 분석
       │
       ▼ Supabase upsert (중복 방지: source+external_id)
```

### 수집 흐름 요약

| 항목 | 내용 |
|------|------|
| **트리거** | 매시간 정각 자동 (`cron: '0 * * * *'`) + 수동 실행 가능 |
| **실행 환경** | GitHub Actions (ubuntu-latest, Node.js 20) |
| **수집 스크립트** | `trend-scouter/scripts/rss-collector.js` |
| **수집 소스** | RSS 피드 6개 (Reddit, Product Hunt, HN, Dev.to 등) |
| **필터링** | 키워드 매칭으로 `impactScore` 계산 → 10점 미만 버림 |
| **AI 분석** | Gemini API로 한국어 헤드라인/요약/PUFE 점수 생성 |
| **저장소** | Supabase (`trends` + `analysis` 테이블, upsert로 중복 방지) |
| **인증 정보** | GitHub Secrets (`NEXT_PUBLIC_SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `GEMINI_API_KEY`) |

### ⚠️ 현재 없는 것

- **GitHub Trending** 소스는 없음 — 처음 아키텍처 문서에서 "GitHub Actions 사용"은 실행 환경을 의미한 것이고, GitHub 데이터를 수집하는 게 아닌 현황입니다.
- GitHub Trending 레포들을 수집 소스로 추가하고 싶으시면 `RSS_FEEDS`에 `https://github.com/trending` 스크래퍼를 별도로 추가해야 합니다.
