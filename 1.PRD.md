# PRD (Product Requirement Document) - Trend Scouter

## 1. 프로젝트 개요 및 목표
웹사이트 또는 앱을 빠르게 개발하여 수익화 가능성을 검증하고, AI를 통해 프로젝트의 지속 여부를 판단하는 체계적인 시스템을 구축하는 것이 목표입니다.

- **핵심 목표**: 다수의 서비스를 신속하게 개발하고, 수익화 가능성을 데이터 기반으로 판단하여 자원을 효율적으로 배분함.
- **AI의 역할**: 서비스 아이디어 도출, 개발 산출물의 목표 부합성 검증, 실제 사용자 행태 분석 및 수익화 가능성 판단.

## 2. 전략적 포지셔닝 및 차별화 (Pivot Strategy)
기존 글로벌 트렌드 서비스(Exploding Topics 등)와의 차별화를 위해 다음 전략을 채택합니다.

- **Hyper-Localization**: 단순 트렌드 정보를 넘어 "글로벌 트렌드를 한국 시장에서 어떻게 실행할 것인가(How-to)"에 대한 실행 가이드를 제공함.
- **Actionable Intelligence**: 정보 제공을 넘어 한국향 비즈니스 복제 및 변형 전략을 제안하는 '지능형 에이전트'로 포지셔닝함.

## 3. 핵심 기능 및 질문에 대한 해답
프로그램 개발부터 마케팅 전략까지 아우르는 전반적인 프로세스를 정의합니다.

- **아이디어 생성 근거**: 글로벌 소스(Reddit, Product Hunt, Indie Hackers 등)의 정량적 데이터(업보트 증가율 등)와 한국 시장의 검색량/경쟁도를 결합하여 도출함.
- **서비스 부합성 및 신뢰도**: AI 스코어링 시 '왜 이 점수가 나왔는지'에 대한 근거(Reasoning Path)를 투명하게 공개하여 신뢰를 구축함.
- **수익화 판단 기준**: 지난 데이터 기반의 트렌드 지속성, 개발 난이도, 한국 내 유사 서비스 존재 여부 및 마이크로 결제($1~5) 전환율 등을 종합적으로 고려함.

## 4. 수익화 및 성장 전략
- **수익 모델**: 고가의 구독 모델 외에도 특정 리포트당 마이크로 결제(Micro-payment)를 도입하여 진입 장벽을 낮춤.
- **리텐션(Retention)**: 단순 정보 열람을 넘어 유저가 아이디어를 검증하고 관리하는 '워크스페이스' 기능을 제공하여 지속적인 방문을 유도함.

## 5. 서버 구성도

서비스의 효율적인 운영과 확장을 위해 다음과 같은 아키텍처를 구성합니다.

```mermaid
graph TD
    User([사용자]) <--> Frontend[Next.js Frontend]
    Frontend <--> Supabase[(Supabase DB/Auth)]
    
    subgraph "Background Processing"
        Scraper[Cloudflare Workers / Scraper] --> Supabase
        Scraper <--> AI[AI Engine - GPT-4o / Claude]
    end
    
    Frontend --> Payments[Stripe / PortOne]
```

### 기술 아키텍처 (Technical Stack)
- **Frontend**: Next.js 14 (App Router) + Tailwind CSS (Vercel 호스팅)
- **Database/Auth**: Supabase (PostgreSQL & GoTrue)
- **Worker (Scraper)**: Cloudflare Workers / GitHub Actions
- **AI Engine**: OpenAI GPT-4o / Claude 3.5 Sonnet
- **Payments**: Stripe / PortOne (토스/카카오)
