# Trend Scouter 프로젝트 개발 계획서 (Project Development Plan)

이 문서는 'Trend Scouter' MVP(Minimum Viable Product)를 4주 안에 개발하고 출시하기 위한 상세 로드맵을 정의합니다.

---

## 1. 프로젝트 목표
- **MVP 출시**: 핵심 기능인 서비스 트렌드 수집 및 AI 분석 대시보드 완성.
- **수익화 검증**: 첫 유료 결제 발생을 위한 결제 연동 완료.
- **자동화 기반 마련**: 추후 확장을 고려한 스카우터 서브에이전트 인프라 구축.

---

## 2. 개발 일정 및 마일스톤 (4주)

### 1주차: 데이터 인프라 및 스크래핑 엔진 구축 (Foundation)
- [ ] **DB 스키마 설계**: Trend, Analysis, User, Subscription 테이블 정의.
- [ ] **Scraper Sub-agent 개발**: 
    - Product Hunt API 연동.
    - Reddit(r/sideproject) RSS/API 연동.
    - Indie Hackers 기본 크롤러 구축.
- [ ] **데이터 파이프라인**: 수집된 원본 데이터를 Supabase에 자동 저장하는 Cron Job 설정.

### 2주차: AI 분석 엔진 및 스코어링 로직 (Intelligence)
- [ ] **AI 분석 프롬프트 엔지니어링**:
    - 비즈니스 모델(BM) 추출 로직.
    - 시장성/개발 난이도 점수 산출 로직.
    - 한국 시장 최적화 전략 생성 루틴.
- [ ] **Batch Processing**: 수집된 데이터를 AI 에이전트가 순차적으로 처리하게 자동화.
- [ ] **데이터 정제**: 유사 아이디어 그룹화 및 중복 제거 알고리즘 적용.

### 3주차: 대시보드 고도화 및 사용자 경험 (Frontend)
- [ ] **인터랙티브 대시보드**: 카테고리별 필터링, 정렬(Score순), 검색 기능.
- [ ] **상세 리포트 페이지**: AI가 분석한 상세 비즈니스 전략 가이드 뷰어 구현.
- [ ] **사용자 인증 시스템**: Supabase Auth를 이용한 간편 로그인(Google/Github) 연동.

### 4주차: 수익화 및 최종 배포 (Monetization & Launch)
- [ ] **결제 시스템 연동**: Stripe 또는 포트원(토스/카카오페이) 연동.
- [ ] **구독/단건 구매 로직**: 결제 상태에 따른 프리미엄 콘텐츠 접근 제어.
- [ ] **Vercel 프로덕션 배포**: 도메인 연결 및 환경 변수 최종 점검.
- [ ] **베타 런칭**: 초기 유저 유입을 위한 마케팅 에이전트 가동(SNS 홍보 자동화).

---

## 3. 핵심 성과 지표 (KPI)
- **데이터 확보**: 주당 100개 이상의 신규 비즈니스 트렌드 수집.
- **AI 신뢰도**: AI 분석 내용의 사용자 정성 평가 80점 이상.
- **전환율**: 방문자 대비 프리미엄 리포트 조회/구매 비중 1% 이상 목표.

---

## 4. 위험 관리 (Risk Management)
- **데이터 차단**: 특정 사이트의 크롤링 차단 시 API 우선 활용 및 프록시 서버 검토.
- **API 비용**: OpenAI/Claude API 사용량 모니터링 및 필요시 비용 효율적인 모델(GPT-4o-mini 등) 혼합 사용.

> [!TIP]
> **"Done is better than perfect."** 
> MVP 단계에서는 화려한 UI보다는 AI가 주는 '정보의 가치'와 '결제 경험'이 정상 작동하는지에 집중합니다.
