# Trend Scouter 프로젝트 개발 계획서 (Project Development Plan)

이 문서는 'Trend Scouter' MVP(Minimum Viable Product)를 4주 안에 개발하고 출시하기 위한 상세 로드맵을 정의합니다.

---

## 1. 프로젝트 목표
- **MVP 출시**: 단순 트렌드 수집을 넘어 '한국 시장용 실행 가젯(How-to)'을 포함한 대시보드 완성.
- **신뢰도 중심의 AI 분석**: 점수의 근거(Reasoning)를 데이터와 함께 제공하여 유료화 설득력 확보.
- **Hyper-Localization**: 글로벌 트렌드를 한국적 맥락으로 재해석한 GTM 전략 제공.
- **수익성 검증**: 첫 유료 결제($1~5 마이크로 결제 및 구독) 발생 및 리텐션 루프 확인.

---

## 2. 개발 일정 및 마일스톤 (4주)

### 1주차: 데이터 소스 다변화 및 수집 탄력성 확보 (Foundations) - [DONE]
- [x] **DB 스키마 설계**: Trend, Analysis, User 테이블 연동 완료.
- [x] **Reddit RSS 연동**: 데이터 수집 및 자동 저장 파이프라인 구축 완료.
- [x] **Product Hunt API 연동**: Developer Token을 통한 데이터 수집 성공.
- [x] **Indie Hackers 스크래핑**: RSS/스크래핑을 통한 데이터 수집 완료.
- [x] **데이터 확보**: 현재 총 56개의 트렌드 데이터 수집 완료 (Reddit 25, PH 10, IH 21).

### 2주차: AI 분석 고도화 및 근거 기반 점수화 (Intelligence) - [DONE]
- [x] **AI 분석 자동화 스크립트 (`scripts/automate_analysis.js`)**: LLM 연동 및 배치 분석 로직 구현 완료.
- [x] **Gemini API 연동**: `GEMINI_API_KEY` 설정 및 Tier 1 업그레이드 완료.
- [x] **AI Reasoning 프롬프트 튜닝**: 점수 부여 근거(Reasoning) 및 한국형 GTM 전략 품질 고도화.
- [x] **Batch Processing**: 수집된 모든 트렌드(99개) 전수 분석 및 저장 완료.

### 3주차: 사용자 가치 체감형 UI/UX & Detail Reports (Experience) - [DONE]
- [x] **대시보드 UI 고도화 (Interactive Dashboard)**:

    - **Hyper-Localization**: 모든 트렌드 제목을 한국어 전문 용어로 번역 및 최적화 완료.
    - `TrendList.tsx` 클라이언트 컴포넌트 분리 및 성능 최적화 완료.
- [x] **심층 분석 상세 페이지 (`/trend/[id]`)**: 
    - `react-markdown`을 활용한 AI 분석 리포트(Reasoning, GTM, Tech Stack) 시각화 완료.
    - 점수 배지 및 시장 지표 디자인 개선 완료.
- [x] **워크스페이스 기능 완성 (Retention)**: 
    - 북마크한 트렌드 목록을 확인하고 관리할 수 있는 사용자 전용 공간 구현 완료.

## 향후 개발 및 운영 계획 (Lean MVP 4단계)

제안하신 "PG 없이 시작하기" 전략에 따라 4주차 개발 계획을 다음과 같이 피봇합니다.

### 1단계: 콘텐츠 검증 (완료 및 공개 준비)
- **현재 상황**: 이미 99개의 해외 AI 트렌드 분석 데이터 확보 완료.
- **액션**: Next.js 프로젝트를 Vercel에 배포하여 대중에게 공개.

### 2단계: 애드센스 승인 + 뉴스레터 구독 수집 (4주차 초반)
- **신규 기능**: 메인 및 상세 페이지에 **이메일 구독 폼** 추가.
- **DB**: `newsletter_subscribers` 테이블을 생성하여 이메일 수집.
- **수익화**: 애드센스 연동을 위한 사이트 최적화.

### 3단계: 유료 전환 테스트 (4주차 후반)
- **신규 기능**: "프리미엄 리포트 구매" 버튼 클릭 시 **카카오페이/토스 송금 링크** 및 QR 코드가 포함된 결제 안내 페이지 노출.
- **수동 승인**: 유저가 입금 완료 알림을 보낼 수 있는 폼 구현. 관리자 수동 확인 후 권한 부여.

### 4단계: 정식 결제 시스템 도입 (유료 구독자 30명 달성 시)
- **향후 과제**: 스텝페이(StepPay) 또는 국내외 PG(Toss/Stripe) 정식 연동. 자동 정기결제 시스템 구축.

---

## 3. 상세 구현 계획 (Implementation Details)

### AI 분석 자동화 파이프라인
- **Process**: `trends` 테이블에서 분석이 없는 항목 추출 -> OpenAI API(gpt-4o-mini) 요청 -> `analysis` 테이블에 정형화된 데이터 저장.
- **Output Columns**: `summary`(Reasoning 포함), `score_revenue`, `score_difficulty`, `score_korea_potential`, `business_model`, `gtm_strategy`, `korea_localization_tips`, `tech_stack_suggestion`.

### UI/UX 개선 사항
- **Card Design**: 난이도, 잠재력 점수 외에 "AI가 추천하는 이유" 한 줄 요약 추가.
- **Detail View**: 탭 구조를 활용하여 [비즈니스 요약 / 시장 분석 / 기술 구현 가이드]로 분리 제공.
