# Trend Intelligence 프로젝트 개발 계획서 (Project Development Plan)

이 문서는 'Trend Scouter' MVP(Minimum Viable Product)를 4주 안에 개발하고 출시하기 위한 상세 로드맵을 정의합니다.

---

## 1. 프로젝트 목표
- **MVP 출시**: 단순 트렌드 수집을 넘어 '한국 시장용 실행 가젯(How-to)'을 포함한 대시보드 완성.
- **신뢰도 중심의 AI 분석**: 점수의 근거(Reasoning)를 데이터와 함께 제공하여 유료화 설득력 확보.
- **Hyper-Localization**: 글로벌 트렌드를 한국적 맥락으로 재해석한 GTM 전략 제공.
- **수익성 검증**: 첫 유료 결제($1~5 마이크로 결제 및 구독) 발생 및 리텐션 루프 확인.

## 향후 개발 및 운영 계획 (Lean MVP 4단계)

기존의 단순 광고/수동 결제 체계에서 벗어나 다음과 같이 수익화 계획을 피봇합니다.

### 1단계: 콘텐츠 검증 (완료 및 공개 준비)
- **현재 상황**: 이미 99개의 해외 AI 트렌드 분석 데이터 확보 완료.
- **액션**: Next.js 프로젝트를 Vercel에 배포하여 대중에게 공개하되, 초기에는 유지보수 최소화 및 SEO 시너지 극대화를 위해 모회사 서브도메인(`trend.gonsuit.com`)을 우선 연결함.

### 2단계: 잠재 고객 수요 확인 (Lean Validation)
- **신규 기능**: 메일 서비스(Newsletter) 등 무거운 기능 개발을 배제함. 대신 주요 GTM 전략 등 핵심 인사이트 하단에 **"더 깊은 분석 자료나 비즈니스 협력 문의하기" (단순 Contact/이메일 발송 폼)** 버튼을 배치하여 유저의 실제 관심도를 측정함.
- **수익화 기초**: 분석 리포트 하단(Tech Stack 등)에 타겟 유저층에 맞는 **제휴 마케팅(Affiliate)** 링크 삽입.

### 3단계: 마이크로 결제(Polar) 테스트 (4주차 후반)
- **신규 기능**: "프리미엄 리포트 전체 보기" 클릭 시 **Polar** 마이크로 결제($1~5) 링크 노출 (기존의 수동 입금 폼 시스템 전략 대체).
- **자동화 인가**: Polar 결제 완료 웹훅(Webhook) 또는 리다이렉트를 통해 해당 유저에게 프리미엄 콘텐츠 접근 권한 즉시 부여.

### 4단계: 정식 정기구독 결제 시스템 확립 (테스트 지표 확보 시)
- **향후 과제**: 초기 마이크로 결제 지표를 분석한 후, **Polar**를 활용한 월간/연간 정기구독(Subscription) 멤버십 모델 런칭 및 자동화 파이프라인 정립.

## 3. 상세 구현 계획 (Implementation Details)

### AI 분석 자동화 파이프라인
- **Process**: `trends` 테이블에서 분석이 없는 항목 추출 -> Gemini API(gemini-2.5-flash-lite) 요청 -> `analysis` 테이블에 정형화된 데이터 저장.
- **Output Columns**: `summary`(Reasoning 포함), `score_revenue`, `score_difficulty`, `score_korea_potential`, `business_model`, `gtm_strategy`, `korea_localization_tips`, `tech_stack_suggestion`.

### UI/UX 개선 사항
- **Card Design**: 난이도, 잠재력 점수 외에 "AI가 추천하는 이유" 한 줄 요약 추가.
- **Detail View**: 탭 구조를 활용하여 [비즈니스 요약 / 시장 분석 / 기술 구현 가이드]로 분리 제공.

## 5. 데이터 수집(Scraping) 프로세스 상세

현재 트렌드 데이터는 `trend-scouter/scripts/` 디렉터리에 위치한 수집용 자동화 스크립트들을 통해 주기적으로 취합되어 데이터베이스에 저장되는 구조입니다.

### 5.1 개별 소스 스크래퍼 (`scripts/scrapers/`)
이 디렉터리에 위치한 모듈들은 각각의 데이터 소스 특성(API, RSS, 웹 크롤링)을 활용하여 원시 트렌드 데이터를 추출합니다.

- **`product_hunt.js`**: Product Hunt의 최신 인기 프로덕트 출시 정보 및 지표 수집
- **`reddit.js`**: Reddit의 주요 하위 커뮤니티(서브레딧)에서 화제가 되는 비즈니스/테크 관련 게시글 수집
- **`indie_hackers.js`**: Indie Hackers 포럼에서 화제가 되는 수익화 달성 팁 및 서비스 런칭 포스트 수집

### 5.2 데이터 취합 및 저장 (`scripts/scouter.js`)
세 가지의 스크래퍼 모듈을 호출하여 데이터를 모으고 가공한 뒤 DB에 안전하게 적재하는 '오케스트레이터' 역할을 수행합니다.

1. **데이터 통합 (Aggregating)**: 내부에서 `fetchProductHuntTrends`, `fetchRedditTrends`, `fetchIndieHackersTrends` 함수를 순차적으로 혹은 비동기로 실행하여 결과값을 하나의 큰 배열로 병합합니다.
2. **중복 검사 및 갱신 (Upsert)**: 취합된 데이터를 Supabase `trends` 테이블에 저장합니다. 이때 게시물의 출처(`source`)와 고유 ID(`external_id`)를 고유 색인으로 사용하여 중복 생성을 막고, 이미 존재하는 데이터라면 최신 지표로 덮어씌웁니다(`onConflict`).
3. **독립적 배치 실행**: 이 스크립트는 백엔드 서버나 프론트엔드 코드에 종속적이지 않고, 독립된 환경(로컬 개발자 PC 혹은 클라우드 크론 잡)에서 실행되며 DB 접근을 위해 `.env.local`의 `SUPABASE_SERVICE_ROLE_KEY`를 사용합니다.

### 5.3 데이터 연동 완전 자동화 (Cloudflare Workers 적용 계획)
현재 수동으로 작동하는 데이터 수집(Scraping)을 매일 정해진 시간(Cron)에 사용자 개입 없이 작동하게 하려면 **Cloudflare Workers**를 세팅해야 합니다. 작업 내역 및 수순은 다음과 같습니다.

1. **Cloudflare 계정 연동 및 의존성 설치**
   - 로컬 터미널에서 `npm install wrangler --save-dev` 설치.
   - `npx wrangler login` 명령어로 개발자의 기기를 Cloudflare 계정과 권한 연동.
2. **Workers 환경 세팅 (`wrangler.toml` 생성)**
   - `trend-scouter` 디렉터리에 `wrangler.toml` 파일을 생성하여 워커 이름 및 설정 정의 (예: `name = "trend-scouter-worker"`, `main = "scripts/scouter.js"`).
   - 예약된 스케줄을 설정하는 트리거 등록. (예: `[triggers] crons = ["0 0 * * *"]` - 매일 자정 실행)
3. **환경 변수 암호화 저장 (Secrets)**
   - 스크래핑 스크립트가 사용할 API 키 보호.
   - `npx wrangler secret put NEXT_PUBLIC_SUPABASE_URL` 및 `npx wrangler secret put SUPABASE_SERVICE_ROLE_KEY` 명령을 통해 민감한 환경변수들을 Cloudflare 서버에 안전하게 주입.
4. **Cloudflare 호환 코드로 변환 (Refactoring)**
   - 현재 Node.js 내장 모듈 단위로 작성된 `scouter.js` 내부 로직을 Cloudflare Workers가 이해할 수 있는 **ES Modules 문법(fetch 이벤트를 수신하거나 export default { scheduled() {} } 형식)**으로 일부 래핑/수정.
5. **원격 배포 (Deploy)**
   - `npx wrangler deploy` 명령어를 실행시켜 코드를 전 세계 Cloudflare 엣지(Edge) 서버로 업로드 및 동작 개시.
